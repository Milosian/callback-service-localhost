## 3️⃣ Common pitfalls & how to avoid them  

| Symptom | Likely cause | Fix |
|--------|--------------|-----|
| **CORS error persists even after adding `cors`** | The middleware is placed *after* the route, or you have multiple `app.use` that terminate the request early. | Move the `app.use(cors(...))` line to the top of the file, right after `const app = express();`. |
| **Only pre‑flight (`OPTIONS`) request succeeds, but `POST` still fails** | The `OPTIONS` response lacks the `Access‑Control‑Allow‑Headers` for `Content-Type`. | In the `cors` options, add `allowedHeaders: ['Content-Type']` (the library adds it automatically, but only if you haven’t overridden defaults). |
| **You need to allow multiple sub‑domains** | You wrote a single string, but requests come from `app.my-site.com` *and* `www.my-site.com`. | Use a function that returns `true` for any allowed origin:  

  ```js
  const allowed = [/\.my-site\.com$/];   // regex array
  app.use(cors({
    origin: (origin, callback) => {
      const ok = allowed.some(rx => rx.test(origin));
      callback(null, ok ? origin : false);
    }
  }));
  ``` |
| **You see the error only in production, not locally** | Your production server is behind a reverse‑proxy (NGINX, Apache, Cloudflare) that strips the CORS header. | Ensure the proxy passes *all* response headers unchanged, or add the header in the proxy config as well (e.g., `add_header Access-Control-Allow-Origin ...;`). |
| **You plan to call the API from a mobile app or server‑side script** | CORS only applies to browsers; you don’t need it for native apps. | You can keep the same server config; the extra header is harmless. |

## 2️⃣ JavaScript that renders the form & calls the API

Create the file **`js/cw-script.js`** inside the plugin folder:

```js
/* cw-script.js – runs in the browser */
(function () {
    // ------------------------------------------------------------------
    // 1️⃣ Helper: simple element creator
    // ------------------------------------------------------------------
    const el = (tag, attrs = {}, ...children) => {
        const node = document.createElement(tag);
        for (const [k, v] of Object.entries(attrs)) {
            if (k === 'class') node.className = v;
            else if (k.startsWith('on')) node.addEventListener(k.slice(2), v);
            else node.setAttribute(k, v);
        }
        children.forEach(ch => {
            if (typeof ch === 'string') node.appendChild(document.createTextNode(ch));
            else if (ch instanceof Node) node.appendChild(ch);
        });
        return node;
    };

    // ------------------------------------------------------------------
    // 2️⃣ Build the form UI
    // ------------------------------------------------------------------
    const renderForm = () => {
        const form = el('form', { id: 'cw-form' },
            el('label', {}, 'Name:',
                el('input', { type: 'text', name: 'name', required: true, placeholder: 'Your name' })
            ),
            el('br'),
            el('label', {}, 'Phone:',
                el('input', { type: 'tel', name: 'phone', required: true, placeholder: '+1‑555‑123‑4567' })
            ),
            el('br'),
            el('label', {}, 'Message (optional):',
                el('textarea', { name: 'message', rows: 3, placeholder: 'Anything else?' })
            ),
            el('br'),
            el('button', { type: 'submit' }, 'Request a callback')
        );

        const resultDiv = el('div', { id: 'cw-result' });

        form.addEventListener('submit', async (e) => {
            e.preventDefault();
            resultDiv.textContent = ''; // clear previous

            const data = Object.fromEntries(new FormData(form));

            // ------------------------------------------------------------------
            // 3️⃣ Send request – direct to your external API (CORS must be open)
            // ------------------------------------------------------------------
            try {
                const resp = await fetch(CW_DATA.apiUrl, {
                    method: 'POST',
                    headers: { 'Content-Type': 'application/json' },
                    body: JSON.stringify(data),
                });

                const json = await resp.json();

                if (json.success) {
                    resultDiv.style.color = 'green';
                    resultDiv.textContent = json.message || 'Request sent!';
                    form.reset();
                } else {
                    resultDiv.style.color = 'red';
                    resultDiv.textContent = json.error || 'Something went wrong';
                }
            } catch (err) {
                resultDiv.style.color = 'red';
                resultDiv.textContent = 'Network error';
                console.error(err);
            }
        });

        return el('div', { class: 'cw-widget' }, form, resultDiv);
    };

    // ------------------------------------------------------------------
    // 4️⃣ Insert the widget into the placeholder <div id="cw-root">
    // ------------------------------------------------------------------
    document.addEventListener('DOMContentLoaded', () => {
        const root = document.getElementById('cw-root');
        if (root) {
            root.appendChild(renderForm());
        }
    });
})();
```

**What this script does**

* builds a small HTML form inside the placeholder created by the shortcode,  
* gathers the fields (`name`, `phone`, `message`),  
* POSTs JSON to the URL you supplied in `CW_DATA.apiUrl`,  
* shows a green success message or a red error message.

> **CORS requirement:** the API (`https://wspace2.great-site.net/api/callback`) must send the `Access-Control-Allow-Origin` header that includes **your WordPress site’s origin** (e.g., `https://www.my‑site.com`). See the previous answer for fixing that on the Node side.

## 4️⃣ Activate the plugin & insert the widget

1. **Log in** to WordPress → **Plugins → Installed Plugins**.  
2. Find **“Callback Widget”** and click **Activate**.  

3. **Edit the page/post** where you want the form to appear and add the shortcode:

```
[callback_widget]
```

That will render a `<div id="cw-root"></div>` placeholder; the JS will replace it with the actual form.

## 6️⃣ Troubleshooting checklist

| Issue | Likely cause | Fix |
|-------|--------------|-----|
| **CORS error** (browser blocks request) | API does not send `Access‑Control‑Allow‑Origin` for your WordPress domain. | Edit the Node server’s `cors` middleware to allow the exact origin (`https://www.my-site.com`). |
| **“Network error”** (no response) | API endpoint unreachable (wrong URL, firewall, or service not running). | Verify the URL in `CW_DATA.apiUrl` matches the live server, and that the server is listening on the correct port. |
| **Email never arrives** | SMTP credentials wrong, or provider blocks the send. | Run the `smtp-test.js` script (from the previous answer) on the same machine that runs the Node API to confirm email works. |
| **Form does not appear** | JS not loaded or `wp_enqueue_script` didn’t fire. | Check page source → look for `<script src="…cw-script.js"></script>` and for the inline `CW_DATA` object. Ensure the shortcode `[callback_widget]` is actually present. |
| **Styling looks broken** | CSS not loaded or conflicts with theme. | Inspect the element in DevTools, confirm `cw-style.css` is loaded, and adjust selectors or use `!important` as a last resort. |
| **You receive the request twice** | Some browsers retry a failed POST; or a page reload after submit. | After a successful submit, the script calls `form.reset();` and does not reload the page. Ensure there is no `action=` attribute on the form (the JS creates the form without it). |

## 7️⃣ Optional: Proxy the request through WordPress (no CORS needed)

If you **cannot** change the CORS header on the external API, you can let WordPress act as a thin proxy:

1. In `callback-widget.php` keep the `admin_post_nopriv_cw_proxy` / `admin_post_cw_proxy` handlers (already included).  
2. Change the JavaScript to POST to `admin-ajax.php` (or the `admin-post.php` URL) instead of directly to the external API:

```js
const proxyUrl = '/wp-admin/admin-post.php?action=cw_proxy';
await fetch(proxyUrl, {
    method: 'POST',
    credentials: 'same-origin',
    body: new URLSearchParams(data)   // sends as application/x-www-form-urlencoded
});
```

3. The PHP handler (`cw_proxy_callback`) forwards the payload to your Node API and returns the JSON back to the browser.  
   This method **bypasses CORS** because the request now stays on the same origin (your WordPress site).

Use this only if you cannot control the Node server’s CORS settings.
